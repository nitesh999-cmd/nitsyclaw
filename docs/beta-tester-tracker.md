# Private beta lead tracker

This is the manual follow-up process for people who request NitsyClaw beta access from `/offer`.

No payment is collected from this flow. It is for demand validation, user interviews, and careful pilot onboarding only.

## Intake fields

Copy each beta request into a private spreadsheet, CRM, or email folder with these columns:

| Field | Why it matters |
|---|---|
| Date received | Measures demand and follow-up speed |
| Name | Human follow-up |
| Email | Reply path |
| Main use case | Finds the strongest paid wedge |
| Source | Shows whether `/offer`, WhatsApp, referral, or direct outreach worked |
| Follow-up status | Keeps every lead moving |
| Safety notes | Records setup, privacy, or risky-action concerns |
| Next action | Makes ownership clear |

## Follow-up status

Use only these statuses until a real CRM is connected:

- New
- Replied
- Booked call
- Pilot accepted
- Not now
- Disqualified

## First reply

Use this plain reply:

```text
Thanks for asking for beta access.

NitsyClaw is currently best for WhatsApp-based reminders, AUD expenses, bill/document summaries, drafts, memory, and proof checks.

Before I invite you, what are the top 2 life-admin jobs you would want it to handle every week?
```

## Safety rules

- Do not onboard paid public users until tenant isolation is verified.
- Do not promise Gmail, Drive, Photos, Spotify, bank feeds, phone calls, SMS sending, or birthdays are live until provider setup is complete.
- Do not collect passwords, bank login details, private access tokens, or sensitive documents by email.
- Keep early pilots supervised and low volume.
- Stop onboarding if WhatsApp proof checks, loop guard, or health checks fail.

## Weekly review

Every week, count:

- Number of new beta requests.
- Number of replies.
- Number of calls booked.
- Top repeated use cases.
- Top blockers.
- Whether any user would pay for the current product without extra integrations.

The goal is not a large list. The goal is 10 serious testers with repeated weekly pain.
